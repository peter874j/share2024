第 1 頁投影片
標題：數位員工開發機採購合理性評估

採購目的：
建置獨立的數位員工 Dev / UAT / CI-CD 開發環境，改善目前開發分散於個人主機與共用主機的問題，並避免開發測試影響正式 K8S 平台。

作法：
- 採購 1 台無 GPU Linux Server，作為數位員工研發交付主機。
- AI 推論仍以閉源 LLM API / Azure API 為主，因此開發機不需 GPU。
- 開發機主要承載 Langflow Dev / UAT、CI/CD、自動測試、API Mock、PostgreSQL / Redis 測試環境、Prompt / Flow 版本驗證、Log / Trace 測試。
- 正式服務仍部署於既有 K8S 平台，開發機僅作為開發、驗證、測試與交付環境。

效益：
- 建立正式平台外的獨立開發環境，降低測試流程影響正式服務的風險。
- 改善目前個人主機與共用主機資源不足、環境不一致、維護困難等問題。
- 支援 10 人團隊進行多專案並行開發、測試與交付。
- 建立標準化 Dev → UAT → CI/CD → 正式部署流程。
- 提升數位員工上線品質，降低正式環境異常與回復成本。
- 不重複投資正式推論平台，既有 K8S 繼續承載正式服務。

應用場域：
- Langflow Flow 開發與測試
- 數位員工 API 串接驗證
- Prompt / Flow 版本控管
- CI/CD 自動部署與回歸測試
- UAT 使用者驗證
- API Mock 與內部系統整合測試
- Log / Trace / 錯誤追蹤測試
- 部門數位員工開發共用環境

版面建議：
左側放一個架構示意：
「開發團隊 → Linux 開發機 → UAT / CI-CD → 既有 K8S 正式平台 → Azure / 閉源 LLM API」
右側用四個區塊呈現：採購目的、作法、效益、應用場域。

第 2 頁投影片
標題：AI Agent 驗證沙盒採購合理性評估

採購目的：
建置部門級 AI Agent / RAG / PoC 驗證沙盒，讓新數位員工需求可先低風險快速試錯，確認具價值後再導入開發機與正式 K8S 平台。

作法：
- 採購 NVIDIA Spark 作為 AI Agent 驗證沙盒設備。
- Spark 不作為正式服務主機，而是用於部門內 PoC、Agent、RAG、Prompt 測試、Open Model 測試與教育訓練。
- 可導入 Nemotron 等開源模型，作為閉源 LLM API 的成本分流機制。
- 採用 Local-first + GPT fallback 架構：
  - 高頻、低風險任務先由 Spark / Nemotron 處理。
  - 複雜推理、高準確度需求或正式回覆再轉送 GPT5.1 / 閉源 LLM API。
- 成熟 PoC 再移轉至 Linux 開發機進行 Dev / UAT / CI-CD，最後部署至既有 K8S 正式平台。

效益：
- 快速驗證 AI Agent、RAG、文件問答、Prompt 與 Flow 的可行性。
- 降低未成熟需求直接進入正式開發造成的資源浪費。
- 分流部分 GPT5.1 / 閉源 LLM API 使用量，降低長期 token 成本。
- 支援多個數位員工 PoC 並行，避免單一測試環境排隊或互相干擾。
- 培養部門自主 AI Agent、RAG、Open Model、本地推論與 Prompt Engineering 能力。
- 保留閉源模型高品質優勢，同時建立開源模型應用能力。
- 支援教育訓練、Demo 展示與跨部門需求初步驗證。

應用場域：
- AI Agent 多步驟任務驗證
- RAG 文件問答與知識查詢
- SOP / API 文件 / 查核規則問答
- 報表摘要與異常分類
- 工單分類與處理建議
- Prompt Engineering 與 Flow 試錯
- Open Model / Nemotron 本地測試
- 教育訓練與主管 Demo
- 閉源 LLM API 成本分流驗證

版面建議：
左側放一個流程圖：
「需求想法 → Spark PoC 沙盒 → 有價值案例 → Linux 開發機 Dev / UAT → 既有 K8S 正式平台」
右側放四個區塊：採購目的、作法、效益、應用場域。
可另外用一個小框標示：
「定位：不是取代 GPT5.1，而是分流高頻低風險任務，重要任務仍回到閉源 LLM API。」

整體結論頁尾或備註可放：
本案調整重點不是重複購置正式推論平台，而是補足數位員工從 PoC、開發、測試到正式上線的交付缺口。正式服務沿用既有 K8S，新增 Linux 開發機與 Spark 沙盒分別支援 Dev / UAT / CI-CD 與 AI Agent / RAG / PoC 驗證，提升部門數位員工落地效率與自主能力。
